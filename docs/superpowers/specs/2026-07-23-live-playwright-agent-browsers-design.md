# ReHoYo Live Playwright Agent Browsers

日期：2026-07-23
状态：已获产品方向批准，等待书面规格复核

## 目标

将当前根据研究事件绘制的浏览器样式卡片替换为真实 Playwright 页面会话。用户应能在 ReHoYo Electron 应用内实时看到四个 Agent 浏览公开网页、执行可审计操作，并在网站显示 Turnstile 或类似挑战时手动接管。任务仍遵守“只使用真实数据”的产品约束：页面无法访问或证据不足时展示缺口，不生成替代评论或虚构结论。

## 范围

本次实现包含：

- 四个 Agent 各自拥有一个隔离的 Playwright 页面池。
- 每个页面池最多四个页面，其中一个页面向 Agent 卡片直播，三个页面在后台运行。
- 全局正常上限为十六个页面：四个直播页面和十二个后台页面。
- Agent 自动执行真实导航、搜索、点击、滚动、等待与可见文本提取。
- Electron 主进程通过受限 IPC 将页面帧、页面状态与可审计操作发送给 React。
- 用户可以展开任一 Agent，查看四个页面标签，固定直播页面，或在挑战状态下接管页面。
- 自适应调度器根据系统资源、页面延迟、限流和挑战状态在四至十六个页面之间调整并发。
- 新页面帧使用仅改变透明度的 120ms 快速淡入；减少动态效果设置下不播放动画。

本次明确不包含：

- CloakBrowser、浏览器指纹伪装、反检测配置或验证码自动破解。
- 绕过 Turnstile、登录墙、付费墙或网站访问控制。
- 保存账号密码、挑战输入、浏览器截图、Cookie 或会话缓存。
- 向用户展示模型隐藏思维链。界面仅展示可观察动作和简短、可审计的任务说明。

## Agent 浏览职责

四个 Agent 都会真实浏览，但职责不同，避免为了视觉效果而重复打开无意义页面。

### 社区研究 Agent

- 发现与版本相关的公开讨论页面。
- 在 Reddit、Niconico、Bilibili、HoYoLAB、米游社及允许访问的公开搜索结果中导航。
- 提取页面标题、URL、公开时间、作者、可见原文和互动量。
- 将通过真实性校验的候选记录交接给下游 Agent。

### 玩家情绪 Agent

- 重新打开候选证据页面并核对上下文，而不是只分析截断摘要。
- 展开被折叠但无需登录即可访问的公开内容。
- 记录原文附近语境并将其交给 GLM 做情绪、主题和中文释义分析。
- 不因单条评论推导全局趋势。

### 地区差异 Agent

- 按 CN、JP、WEST 分配页面，核对语言与来源归属。
- 对同一主题执行跨地区证据比对，并明确缺少样本的地区。
- 只从已打开并验证的页面派生地区计数。

### 策略建议 Agent

- 重新打开最终引用页面，对争议和建议所依赖的证据做最后核验。
- 检查引用 URL、原文与报告证据编号是否一致。
- 搜索可证伪信号；无法获得足够证据时删除相应结论。

## 架构

### Electron 主进程

新增 `BrowserSessionManager`，由 Electron 主进程持有，渲染进程不能直接访问 Playwright 对象。每个研究任务建立一个临时浏览器上下文，并在上下文内建立四个 `AgentBrowserPool`。每个池最多维护四个页面和一个可切换的直播页面标识。

Playwright 使用隐藏浏览器运行，不打开外部浏览器窗口。运行时依赖作为正式应用依赖打包，并通过明确的浏览器可执行文件发现策略启动。启动失败时任务展示真实错误，不退回静态浏览器动画。

### 页面工作器

每个 `BrowserWorker` 维护以下状态：

- `queued`
- `navigating`
- `interacting`
- `extracting`
- `challenge_waiting`
- `human_control`
- `resuming`
- `completed`
- `failed`

工作器只接受经过主进程校验的 HTTPS 公网 URL。导航前解析主机与目标地址，拒绝 `file:`、`data:`、`javascript:`、回环地址、链路本地地址和私有网络地址。下载被取消；新窗口在经过相同校验后转换为池内页面，否则关闭。

### IPC 协议

主进程向渲染进程发送：

- `FRAME`：会话、Agent、页面、序号、时间、尺寸、URL 和压缩截图。
- `ACTION`：`NAVIGATE`、`CLICK`、`SCROLL`、`TYPE`、`EXTRACT`、`WAIT`、`CHALLENGE`、`RESUME`。
- `TAB`：页面列表、标题、URL、状态与当前直播页面。
- `STATUS`：页面池和浏览器会话状态。
- `CHALLENGE`：挑战类型、页面标识和是否允许人工接管。
- `METRICS`：活跃页面数、排队数、延迟与当前并发上限。

所有消息均包含 `runId`，渲染进程只接收当前任务消息。IPC 输入需要运行时校验、长度限制和 URL 校验。截图限制像素尺寸和单帧字节数，旧帧可丢弃，避免 IPC 背压阻塞研究任务。

渲染进程向主进程发送：

- 选择或固定直播页面。
- 开始或结束人工接管。
- 映射后的点击坐标、滚动增量和有限键盘输入。
- 跳过当前挑战页面。

正常 Agent 控制状态不接受用户输入；只有 `human_control` 状态允许输入命令。

## 实时画面与交互

每个 Agent 卡片都有一个真实直播区域，显示当前页面截图、真实 URL、标题、页面状态、最近动作和进度。直播默认自动跟随该 Agent 最近执行动作的页面。用户固定某个页面后，自动跟随暂停，直到解除固定。

点击卡片打开扩展面板。扩展面板包含：

- 四个真实页面标签及各自状态。
- 当前页面的较大 16:9 预览。
- 可审计动作列表。
- 当前并发、页面延迟与排队数量。
- 挑战状态下的“接管验证”和“跳过来源”操作。

截图更新采用双缓冲：新图片加载完成后，以 120ms 将其透明度从 0 变为 1。不得使用位移、缩放或页面整体动画。`prefers-reduced-motion: reduce` 时直接替换帧。活动页面最高约 2 FPS；无操作时只在页面或状态变化后发送一帧。

“操作说明”描述可观察行为，例如“打开 Reddit 讨论页”“向下滚动寻找更多公开评论”“提取三条可见回复”。不得使用“思维链”“内部推理”或不可验证的心理活动措辞。

## 人工挑战接管

页面标题、URL、可见文本或挑战 iframe 表明存在 Turnstile/类似挑战时，工作器进入 `challenge_waiting`，停止该页面上的 Agent 操作，并发送 `CHALLENGE` 事件。其他页面与 Agent 继续运行。

用户选择接管后进入 `human_control`：

- 点击坐标按截图与实际 viewport 比例映射。
- 滚轮和方向输入转发到当前页面。
- 键盘输入只在明确聚焦的页面控件中发送。
- 不记录输入内容，不将其加入 Timeline，也不保存到日志或报告。

当挑战元素消失、页面成功导航或应用检测到目标内容可访问时，状态变为 `resuming`，生成一条不含用户输入的 `RESUME` 事件，并恢复 Agent。十分钟内未完成时页面标记失败并提供跳过选项；系统不尝试绕过。

## 自适应调度

调度器目标上限为十六个页面，最低保留每个 Agent 一个页面。它每两秒评估：

- 可用系统内存。
- 页面操作与截图耗时。
- IPC 待发送帧数量。
- 同域名限流或挑战数量。
- 活跃与排队工作量。

健康状态下逐步提升到每个 Agent 四个页面。出现资源压力时按 `16 → 12 → 8 → 4` 逐级降低后台并发，优先保留四个直播页面。恢复需要连续健康窗口，避免频繁升降。

每个域名还具有独立并发和最小请求间隔。限流、挑战或连续导航失败只暂停对应域名，不降低无关来源的工作能力。自适应变化通过 `METRICS` 和 `ACTION` 事件明确展示。

## 证据与数据完整性

浏览器截图只是实时观察界面，不是报告证据。证据仍必须包含：

- HTTPS 公开 URL。
- 页面真实标题与可见原文。
- 抓取时间。
- 来源、地区与语言。
- `synthetic: false`。

证据在浏览器提取、GLM 分析、渲染接收和本地存储四个边界重复校验。报告计数由最终证据集合确定性派生。页面加载失败、挑战未完成或内容无法核实时不生成证据。截图、浏览器动作或模型输出不能单独变成玩家评论。

## 生命周期与隐私

页面帧保存在渲染进程的有限内存环形缓冲中，新帧淘汰旧帧。任务完成、失败、离开工作区或应用退出时：

- 中止尚未完成的页面动作。
- 关闭 Playwright 页面、上下文和浏览器进程。
- 清除 Cookie、缓存、截图、挑战输入与临时页面数据。
- 只保留真实证据、可审计动作摘要和报告所需事件。

应用日志不得包含 API key、Cookie、挑战输入或完整截图数据。

## 故障处理

- 浏览器启动失败：任务失败并提示缺少浏览器运行时，不显示静态假预览。
- 单页面崩溃：在同一池中最多重建一次；再次失败则跳过该来源。
- IPC 背压：丢弃过期截图，保留最新帧和所有状态/动作事件。
- 页面超时：记录失败并释放 worker，其他页面继续。
- 证据校验失败：丢弃记录并在 Timeline 显示完整性错误。
- 挑战超时：等待用户选择重试或跳过，不执行自动绕过。
- 应用路由离开或任务取消：立即销毁会话。

任何故障都不得触发模拟数据回退。

## 测试策略

### 单元测试

- 页面状态机与合法状态转换。
- IPC 消息运行时校验、消息大小和 URL 限制。
- 私有网络及危险协议拒绝。
- 直播页面自动跟随与用户固定逻辑。
- 调度器在资源压力下按 `16 → 12 → 8 → 4` 降载并稳定恢复。
- 挑战检测、人工接管、自动恢复和十分钟超时。
- 截图环形缓冲和背压丢帧策略。

### Playwright 集成测试

使用本地可控 HTTPS 测试站点验证真实导航、点击、滚动、输入、内容提取、截图和多标签调度。使用本地挑战夹具模拟“需要人工操作后内容解锁”的状态，但不模拟或测试真实验证码绕过。

测试必须先失败，再实现对应生产行为。生产代码不能依赖测试夹具。

### Electron 端到端测试

- 四个 Agent 均显示真实页面帧，而非证据摘要占位图。
- 每个扩展面板显示四个页面标签。
- 正常目标为十六页面，其中四个直播、十二个后台。
- 模拟资源压力时 UI 显示自适应降载。
- 人工接管中的点击和滚动能够改变真实测试页面。
- 挑战解除后 Agent 自动继续。
- 新帧只使用 120ms opacity 动画；减少动态效果时动画关闭。
- 1440×900 与 1920×1080 无遮挡、溢出和控制台错误。

### 真实联网冒烟

仅在明确配置环境中访问少量公开 HTTPS 页面，验证浏览器能够启动、导航并产生带 URL 的真实证据。测试不依赖固定评论数量，也不尝试解决或绕过挑战页。

## 验收标准

- App 外部不出现浏览器窗口。
- 四个 Agent 卡片均来自真实 Playwright 页面帧。
- 正常资源下可达到十六页面，界面明确显示四个直播与十二个后台页面。
- 资源或网站压力下自动降低并发且不中断其他可运行来源。
- 用户可以固定直播页面，并在挑战时于 App 内接管。
- 挑战完成后 Agent 自动恢复；未完成时不绕过、不伪造数据。
- 所有报告数据仍可追溯到真实 HTTPS 证据。
- 截图和用户输入不会持久化。
- 所有新增单元、集成、Electron E2E、TypeScript 与生产构建检查通过。
